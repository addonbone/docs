#### 1. Get Started

- **Назначение**: Служить точкой входа для новых пользователей, помогая быстро начать работу с Addon Bone.
- **Подразделы**:
  - **Introduction**:
    - **Содержание**: Краткий приветственный обзор Addon Bone (например, "Фреймворк на базе Vite для расширений браузера"), его цели (например, простота, поддержка разных браузеров) и целевой аудитории (например, разработчики JavaScript).
    - **Фокус разработчика**: Подчеркнуть уровень подготовки (например, базовые знания Vite будут полезны) и ключевые преимущества (например, поддержка TypeScript).
    - **Лучшие практики**: Содержание до 150 слов, использование изображения-героя или логотипа, ссылка "Следующие шаги" на Installation.
  - **Installation**:
    - **Содержание**: Пошаговые инструкции (например, npm install -g @addonbone/cli, addonbone init), системные требования (например, Node.js 18\+), советы по устранению неполадок (например, "Очистите кэш npm").
    - **Фокус разработчика**: Указать на возможные конфликты (например, существующие конфигурации Vite) и проверить установку командой (например, addonbone --version).
    - **Лучшие практики**: Использование нумерованных списков, шаг проверки, альтернативные методы установки (например, Yarn, pnpm). Добавить раздел "Частые проблемы".

#### 2. Essentials

- **Назначение**: Предоставить базовые знания и практические рекомендации по ключевым функциям Addon Bone.
- **Подразделы**:
  - **Structure**:
    - **Содержание**: Объяснение структуры проекта (например, /assets, /public, addonbone.config.ts) с диаграммой или представлением дерева файлов.
    - **Фокус разработчика**: Предупреждение о модификации важных файлов (например, manifest.json напрямую) и рекомендация использовать команды addonbone.
    - **Лучшие практики**: Пример структуры файлов, ссылки на Assets и Environment Variables, использование сворачиваемых деталей для продвинутых заметок.
  - **Configuration**:
    - **Содержание**: Подробности опций addonbone.config.ts (например, targetBrowsers, manifest) с примерами и значениями по умолчанию.
    - **Фокус разработчика**: Подчеркнуть обязательные и необязательные конфигурации, тестировать изменения с addonbone build --watch.
    - **Лучшие практики**: Использование вкладок для конфигураций на разных языках (TS/JS), предоставление "Чек-листа валидации конфигурации".
  - **Entrypoints**:
    - **Содержание**: Объяснение типов точек входа (например, Background, Content Script) и способов их определения, с примерами кода.
    - **Фокус разработчика**: Указать на особенности точек входа для разных браузеров (например, ограничения Safari MV3) и использование defineContentScript.
    - **Лучшие практики**: Таблица типов точек входа, ссылки на конкретные страницы (например, Background), совет "Тестирование точек входа".
  - **Extension APIs**:
    - **Содержание**: Обзор API браузера (например, chrome.runtime, [browser.storage](http://browser.storage)) с примерами использования.
    - **Фокус разработчика**: Предупреждение о доступности API (например, MV2 vs. MV3) и рекомендации по полифиллам при необходимости.
    - **Лучшие практики**: Ссылки на документацию MDN, интерактивные примеры, заметки об устаревших функциях.
  - **Storage**:
    - **Содержание**: Руководство по использованию API хранения (например, [browser.storage](http://browser.storage).local) с примерами настройки и синхронизации.
    - **Фокус разработчика**: Подчеркнуть лимиты квот (например, 10MB в Chrome) и проблемы с сохранением данных.
    - **Лучшие практики**: Добавить подраздела "Лучшие практики" (например, шифрование чувствительных данных), примеры обработки ошибок.
  - **Messaging**:
    - **Содержание**: Объяснение передачи сообщений (например, browser.runtime.sendMessage) с кодом отправителя и получателя.
    - **Фокус разработчика**: Указать на производственные накладные расходы при частых сообщениях и рекомендовать пакетную обработку.
    - **Лучшие практики**: Полный пример (например, Content Script к Background), ссылка на Relay, если применимо.
  - **Locales**:
    - **Содержание**: Руководство по интернационализации (например, настройка \_locales, файлы сообщений) с примером перевода.
    - **Фокус разработчика**: Предупреждение о языках по умолчанию и тестирование с addonbone build --locale.
    - **Лучшие практики**: Чек-лист "Локализация", ссылки на инструменты i18n.
  - **Icon**:
    - **Содержание**: Инструкции по добавлению иконок (например, icons в манифесте), с требованиями к размеру (например, 128x128px).
    - **Фокус разработчика**: Указать на специфические требования к иконкам для разных браузеров (например, высокое разрешение Firefox).
    - **Лучшие практики**: Предоставить шаблон иконки для скачивания, включить шаг "Предпросмотр".
  - **Assets**:
    - **Содержание**: Объяснение использования /assets и /public, с примерами (например, import from '~/assets/image.png').
    - **Фокус разработчика**: Предупреждение о доступности в content scripts и использование browser.runtime.getURL.
    - **Лучшие практики**: Вкладки для примеров на JS/HTML/CSS, раздел "Советы по производительности".
  - **Environment Variables**:
    - **Содержание**: Подробности использования .env файлов (например, ADDONBONE_API_KEY), с примерами доступа во время выполнения.
    - **Фокус разработчика**: Подчеркнуть правила префиксов (например, ADDONBONE\_) и безопасность (например, избегать коммита .env).
    - **Лучшие практики**: Таблица встроенных переменных, ссылка на Configuration.
  - **Plugin**:
    - **Содержание**: Руководство по созданию пользовательских плагинов (например, definePlugin) с базовым примером.
    - **Фокус разработчика**: Указать на хуки жизненного цикла плагинов и совместимость с Vite.
    - **Лучшие практики**: Чек-лист "Разработка плагинов", ссылка на Resources/Plugins.
  - **Publishing**:
    - **Содержание**: Шаги создания ZIP (например, addonbone zip), отправки в магазины и обновлений версий.
    - **Фокус разработчика**: Предупреждение о политиках магазинов (например, ограничения Chrome на YouTube) и тестирование ZIP локально.
    - **Лучшие практики**: Чек-лист "Подготовка к отправке", ссылки на руководства магазинов.
  - **Специфические страницы точек входа (Background, Command, Service, Content Script, Relay, Offscreen, Page, Popup, Sidebar)**:
    - **Содержание**: Подробное руководство для каждого типа точки входа (например, жизненный цикл Background, инъекция Content Script), с кодом и конфигурацией манифеста.
    - **Фокус разработчика**: Подчеркнуть время run_at, последствия service worker для MV3 и методы тестирования.
    - **Лучшие практики**: Использование единого шаблона (Обзор, Настройка, Пример), ссылки на Entrypoints, сценарии использования.

#### Общие лучшие практики для всех страниц

- **Согласованность**: Использовать H1 для заголовков страниц, H2/H3 для разделов, блоки кода с тегами языка (например, \`\`\`typescript:disable-run
- **SEO/доступность**: Добавить метаописания (например, "Узнайте установку Addon Bone"), альтернативный текст для изображений, метки ARIA.
- **Версионность**: Указать версию и дату (например, "Обновлено для v1.5.0, 14 июля 2025"), ссылка на changelog.
- **Обратная связь**: Добавить кнопку "Было ли это полезно?" или ссылку на отчет об ошибках (например, GitHub).
- **Производительность**: Оптимизировать с ленивой загрузкой изображений, минимальным JS.
- **Локализация**: Планировать i18n с заполнительным текстом или инструментами.
