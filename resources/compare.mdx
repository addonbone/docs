# Compare

This page provides a detailed comparison of the **Addon Bone**, **WXT**, and **Plasmo** frameworks for building browser extensions. Each framework offers unique features and benefits, making them suitable for different use cases. Below, you'll find an overview and a side-by-side comparison to help you choose the right tool for your project.

## Overview

A quick summary of each framework:

| Framework      | Language Support  | Build Tool   | Dev Experience          | Output Size   | Type Safety | Actively Maintained |
| -------------- | ----------------- | ------------ | ----------------------- | ------------- | ----------- | ------------------- |
| **Addon Bone** | TypeScript, React | Rspack       | Fast, Configurable      | 🟢 Very Small | ✅ Strong    | ✅ Yes               |
| **WXT**        | TypeScript        | esbuild/Vite | Minimal, Fast           | 🟢 Small      | ✅ Strong    | ✅ Yes               |
| **Plasmo**     | TypeScript        | esbuild      | DX focused, opinionated | 🟡 Medium     | ✅ Strong    | ✅ Yes               |

### Addon Bone

Addon Bone is a modern framework designed to simplify the development of browser extensions with a focus on flexibility. It provides a streamlined workflow for managing assets, environment variables, and cross-browser compatibility.

### WXT

WXT is a next-generation framework built on Vite, offering an intuitive and safe way to develop web extensions. It emphasizes a conventional structure, automatic imports, and support for multiple browsers, making it a popular choice for developers.

### Plasmo

Plasmo is an all-in-one platform that simplifies the creation, testing, and publishing of browser extensions. It provides built-in support for React, Vue, and Svelte, along with features like Content Scripts UI and automated deployment.

## Detailed Insights

### When to Use Addon Bone

- Ideal for developers who need flexibility with various UI frameworks and prefer a Vite-based workflow.
- Best for projects requiring custom configurations and asset management.

### When to Use WXT

- Suitable for developers seeking a conventional structure with minimal setup.
- Great for projects benefiting from Vite's performance and smaller build sizes.

### When to Use Plasmo

- Perfect for React, Vue, or Svelte users who want an all-in-one solution with built-in deployment.
- Recommended for projects needing quick prototyping and UI injection features like CSUI.

## Conclusion

The choice between Addon Bone, WXT, and Plasmo depends on your project's requirements, preferred UI framework, and deployment needs. Explore their documentation and examples to find the best fit for your workflow.
